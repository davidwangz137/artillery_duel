# AGENTS.md

Instructions for coding agents working in this repository.

## Commit attribution

Agent-authored commits must include the co-author trailer so attribution stays
consistent across agent PRs:

```
Co-Authored-By: OMP Agent <omp-agent@users.noreply.github.com>
```

Commit author email doesn't affect GitHub review identity (reviews still require
a non-author account/token), but the trailer keeps commit attribution consistent.

## Agent-generated reviews and comments

When an agent posts through a human-owned GitHub token, prefix the content with a
role tag so the web UI makes attribution explicit (the GitHub account is the
transport identity, but the content was generated by an agent). Pick the tag
matching the agent's role:

- `[agent coder output]` — the agent is the **author/coder**: responding to
  review feedback, or posting implementation notes/status on its own changes.
- `[agent reviewer output]` — the agent is the **reviewer**: posting review
  feedback or review comments on someone else's changes.
